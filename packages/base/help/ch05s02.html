5.2. Selecting an Image Loading Method

Once you know your images' properties, you can proceed to load them. Corelyzer offers two methods.

The first method, Load local image files, is recommended for most users. You'll select a batch of image files to load. Corelyzer will attempt to organize them into tracks based on their filenames. You can modify this order as needed, along with other image properties like resolution (DPI) and orientation. This approach works well when your images' filenames use the LacCore, IODP, or ICDP section naming convention or similar. Loaded cores in each track will be placed end to end.

If your cores do not use a standard naming convention, it may be easier to use the second method, Load images with an Image Listing file. For each track, you'll need to create a CSV file with the image file and properties for each section in that track. This method also allows you to position each core at a specific depth.
