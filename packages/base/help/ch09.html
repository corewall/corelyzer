9.1. Save and Load Sessions on the Local Machine

It is critical to understand that a saved session file does not include images and dataset files. Rather, it saves the locations of those files on the local machine so they can be loaded along with the session. This means a session file will only function on the local machine where it was created!

You can save your current session by selecting the File > Save Session or File > Save Session as... menu item. This saves the session as a XML-based CML (for "Core Modeling Language") file. The complete CML file schema description can be found here.

A saved session can be loaded with the File > Open Session... menu item.

To copy a complete Corelyzer session to another machine or share with a colleague, use the Core Archive function, which creates a single file containing all resources (images, data files, etc.) suitable for transfer to other machines.
